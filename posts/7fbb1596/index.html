Ubuntu 配置 Laravel + Laradock

Created2022-10-06|Updated2022-10-07|DevelopmentLaravel
|Post View:

前置作業

使用 Ubuntu 22.04

  1. 更新 Ubuntu 套件

    sudo: 讓使用者以擁有更高權限的 superuser 身分執行指令。
    apt: Ubuntu 的套件管理工具。

  2. 安裝 Vim 文字編輯器

    -y: yes, 安裝過程中跳出的提示都以 yes 帶過。

建立 Laravel 空白專案

ref: https://laravel.com/docs/9.x/installation

前置作業

要建立 Laravel 專案前要先:

  1. 安裝 PHP
  2. 安裝 Composer
    ref: https://getcomposer.org/download/
  3. 查看 PHP 及 Composer 版本順便確認安裝成功

建立空白專案

  1. 安裝建立過程會用到的相關套件
  2. 建立一個 blog 空白專案
  3. 查看 Laravel 版本
    Artisan Console
    在建立空白專案過程中可能會出現一些錯誤，基本上就是 1. 裡的套件沒有安裝到

安裝 Laradock

ref: https://laradock.io/introduction/
Laradock 是一個用 Docker 建立的 PHP 開發環境，提供許多常見的功能: PHP 版本切換、database、web server …
不知道 Dcoker 是什麼也沒關係，先用再說。就如官網說的: Use Docker First - Learn About It Later!

前置作業

  1. 安裝 Git
  2. 安裝 Docker
    ref: https://docs.docker.com/engine/install/ubuntu/
    以下指令跟官網一樣 運行 docker hello-world 確認安裝成功

Clone Laradock

在 blog 專案下 clone Laradock

目錄結構：

Laradock 運行 Laravel

  1. 先到 laradock 目錄下複製一份 .env
  2. 將 .env 裡的 PHP_VERSION 改為 8.1
  3. 運行 nginx container 最後輸入網址就可以成功看到首頁。

因為 Compoer 需要的 PHP 版本不同，有可能輸入網址後會得到下列訊息:
Composer detected issues in your platform: Your Composer dependencies require a PHP version ">= 8.1.0".
處理方法為：

  • 跟上面 2. 一樣，先將 PHP 改為指定版本 (這邊必須 >= 8.1.0)
  • rebuild php-fpm: sudo docker compose build php-fpm
  • restart php-fpm: sudo docker compose up --force-recreate -d php-fpm
